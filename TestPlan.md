# Test Plan – Banking Application (BFSI)

## 📌 Objective

To validate the core functionalities of a banking web application including login, account operations, fund transfer, and transaction processing.

---

## 🏦 Scope

The following modules are covered under testing:

* Login Module
* Account Balance
* Fund Transfer
* Transaction History
* Security (Unauthorized Access)

---

## 🧪 Testing Types

* Functional Testing
* Validation Testing
* Security Testing
* Performance Testing

---

## 🔍 Test Scenarios

* Verify login with valid and invalid credentials
* Validate account balance display
* Test fund transfer with valid and insufficient balance
* Check transaction history records
* Verify unauthorized access restriction
* Identify performance delays during transactions

---

## 🛠️ Tools Used

* Browser (Chrome)
* Excel (CSV for Test Cases & Bug Reports)

---

## 📊 Deliverables

* Test Cases Document (CSV)
* Bug Report Document (CSV)
* Screenshots (Proof of testing)

---

## ⚠️ Assumptions

* Demo data is used for testing
* No real transactions are performed
* System behavior is based on UI response
